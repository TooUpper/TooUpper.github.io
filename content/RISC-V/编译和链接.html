编译和链接

GCC

GCC 简介

  • GCC（GNU Compiler Collection）
    • https://gcc.gnu.org/
    • 由 GNU 开发的，遵循 GPL 许可证发行的编译器套件。
    • 支持 C、C++、Objective-C、Fortran、Ada 和 Go 语言等多种语言前端，已被移植到多种计算机体系架构上，如 x86、ARM、RISC-V 等。
    • GCC 的初衷是为 GUN 操作系统专门编写一款编译器，现已被大多数 “Unix-like” 操作系统（如 Linux、BSD、MacOS等）采纳为标准的编译器。

GCC 的命令格式

  • gcc [options] [filenames]
    • -E ：只做预处理(在 C语言 中就是把 # 开头的 宏语言 转为标准的 C语言)
    • -c：只编译不链接，生成目标文件”.O”。(编译是将C语言转为对应的机器指令，链接是最后生成可执行程序的时候，将.O文件和其他的一些库文件链接起来。)
    • -S：生成汇编代码。（C -> 汇编 -> 机器指令）
    • -o file：把输出生成到由 file 指定文件名的文件中。
    • -g：在输出的文件中加入支持调试的信息 (只有加了，程序才可以进行调试，不然不可以)。
    • -v：显示输出详细的命令执行过程。

GCC 的主要执行步骤

  • 编译(cc1，这里针对 C语言，不同的语言有自己的编译器)：编译器完成 “预处理” 和 “编译”。“预处理” 指处理源文件中以 “#” 开头的预处理指令，譬如 #include、#define等。“编译” 则针对预处理的结果进行一系列的词法分析、语法分析、语义分析，优化后生成汇编指令，存放在.O为后缀的目标文件中。
  • 汇编(as)：汇编器将汇编语言代码转换为机器(CPU)可以执行的指令。
  • 链接(ld)：链接器将汇编器生成的目标文件和一些标准库 (譬如 libc) 文件组合，形成最终可执行的应用程序。

GCC 涉及的文件类型

  • .c：C 源文件
  • .cc / .cxx / .cpp：C++ 源文件
  • .s / .S：汇编语言源文件（大 S 的也会包含一些 # 开头的预处理指令，小 S 就是纯的汇编指令）
  • .h：头（header）文件
  • .o：目标（object）：文件
  • .a / .so：编译后的静态库（archive）文件和共享库（shared object）文件
  • a.out：可执行文件

EFL 简介

  • EFL（Executable Linkable Format）是一种 Unix-like 系统上的二进制文件标准格式

EFL 文件格式

  • ELF Header：包含该文件的一些基本信息，如：文件运行在哪种体系架构上的，版本号信息等。
  • Program Header Table：存放运行时信息（运行试图）
  • .text：放一些程序的指令
  • .init：初始化指令
  • .data：放一些数据，全局变量

  • Section Header Table：定义了 EFL 文件中有那些 节(上面的 白色的 Section ) ，节有多大，在什么位置，都通过该文件进行描述；从链接的角度去描述了文件的内容。

运行视图：当 EFL 加载到内存时，如果对于一个个 结 不加区分的分别加载进内存可能会引入一个问题：(操作系统在加载 节 的时候，会按照一定的宽度进行对齐；假设系统按照 4k 进行对齐，text 这个 节 大小只有 40 个字节，但是他也会占一个 4k 大小的区域，不会再放其他东了，那么一个本来很小的程序，展开后就会变得很大，造成空间的浪费。所以真正在加载到内存中的时候会根据 节 的属性来做一些归并，如 text 和 init 节 中放的都是一些可执行程序指令，那么就将二者放在一起继续归并称之为 段(Segment)；所以在加载入内存的时候，实际是按段进行加载的，每个段中包含一个或多个 节(Secction)，因为上述的内容是在运行时才会用到的，所以称之为：运行视图)。

EFL 文件处理相关工具：Binutils

这些 EFL 文件格式如果手动去分析内容会非常的麻烦，所以 GNU 提供了一些配套的工具软件binutils

  • https://www.gun.org/software/binutils
    • ar：归档文件，将多个文件打包成一个大文件。
    • as：被 gcc 调用，输入汇编程序，输出目标文件供链接器 ld 连接。
    • ld：GUN 链接器。被 gcc 调用，他把目标文件和各种库文件结合在一起，重定位数据，并链接符号引用。
    • objcopy：执行文件格式转换。
    • bojdump：显示 EFL 文件的信息。
    • readelf：显示更多 EFL 格式文件的信息（包括 DWARF 调试信息）
